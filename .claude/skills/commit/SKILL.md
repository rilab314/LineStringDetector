# Commit Skill

사용자가 "커밋해줘" 또는 커밋을 요청하면 아래 절차를 따른다.

## 절차

1. `git status`와 `git diff HEAD`를 실행해 변경사항을 파악한다.
2. 아래 규칙에 따라 커밋 메시지를 **한 줄**로 작성하여 사용자에게 보여준다.
3. 사용자가 승인하면 `git add -A` 후 커밋한다. 수정 요청 시 메시지를 수정 후 재확인한다.

## 커밋 메시지 형식

```
(파일명1) 변경사항1 / 변경사항2 // (파일명2) 변경사항3
```

- **영어**로 작성한다.
- 파일명은 확장자 없이 베이스네임만 사용한다 (예: `lane_detector`).
- 여러 변경사항은 `/`로, 파일 간 구분은 `//`로 연결한다.
- **주요 알고리즘·로직 변화만** 기록한다.
- 단순 파라미터 값 변경, 경로/파일명 변경, 포맷 정리, 오타 수정 등 자잘한 변경만 있는 파일은 **메시지에서 생략**한다.
- 삭제된 파일은 `remove <기능>` 형태로 기록한다.
- 새 파일은 `add <기능>` 형태로 기록한다.

## 예시

```
(lane_detector) add endpoint merging step / refactor thinning pipeline // (evaluator) fix AP threshold calculation
```

## 주의사항

- `git add -A`는 반드시 사용자 승인 **후에** 실행한다.
- 커밋 메시지에 `Co-Authored-By` 등 부가 정보를 **추가하지 않는다**.
- 커밋 메시지는 **단일 `-m` 인자**로 전달한다.
